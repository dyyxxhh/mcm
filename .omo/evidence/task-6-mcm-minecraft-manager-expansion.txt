### Schema design
```rust
pub struct McmPackage {
    schema_version: u32,           // currently only 1
    name: String,                  // normalized [a-z0-9-], 1-64, no reserved names
    version: String,
    description: Option<String>,
    game_version: Option<String>,
    loader: Option<String>,
    dependencies: Vec<Dependency>,            // {name, version_requirement?}
    mods: Vec<ModEntry>,                      // {logical_id, provider, project_id, file_id, version, filename, sha256?, download_url?}
    shaderpacks: Vec<Asset>,                  // {path, source: embedded|referenced, hash?, size?}
    resourcepacks: Vec<Asset>,
    datapacks: Vec<Asset>,
    saves: Vec<Asset>,
    configs: Vec<Asset>,
    actions: Option<Vec<Action>>,             // {name, kind: shell, command, cwd?} — Linux shell only
    launch: Option<LaunchRequest>,            // {game?, args}
    local: Option<LocalPrivate>,              // {settings: BTreeMap, history: Vec} — excluded from public export
}
```

### Boundary parser enforcement (parse_mcm_package)
1. **Size limit**: JSON > 10MB → `bail!("package JSON exceeds {MAX_JSON_BYTES} bytes")`
2. **Depth limit**: nesting > 64 → `bail!("package JSON nesting depth {depth} exceeds {MAX_DEPTH}")`
3. **Secret scan**: recursively scans deserialized `serde_json::Value` BEFORE typed parse; any key (case-insensitive) containing `token`/`secret`/`password`/`credential`/`api_key` → `bail!("package contains secret field: {key}")`
4. **Schema version**: field `schema_version: u32`; only version 1 supported; unknown → `bail!("unsupported schema version {X}")`
5. **Package name normalization**: lowercase `[a-z0-9-]`, 1-64 chars, alphanumeric start/end, no consecutive hyphens, no reserved names (`mcm` + Windows reserved: `con`/`prn`/`aux`/`nul`/`com1-9`/`lpt1-9`)
6. **Asset path traversal**: rejects empty, null bytes, `..`, absolute paths (`/`), backslashes, Windows-reserved path components
7. **No raw `serde_json::Value` in domain logic**: parsed once at boundary into typed `McmPackage`; the only opaque `Value` is in `LocalPrivate` (settings/history) which domain logic never interprets

### CLI surface: `pkg info <path>`
Reads `.mcm` file → `parse_mcm_package` → prints normalized summary (name, version, description, game_version, loader, schema_version, counts of all asset types, actions/launch/local presence). Read-only — installs nothing.
