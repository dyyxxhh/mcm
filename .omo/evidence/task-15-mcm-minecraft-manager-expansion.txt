Architecture notes
------------------
1. SourceStore (src/server/source_store.rs) is a filesystem-backed read-only
   store. It reads data_dir/source-index.json and data_dir/source-blobs/<slug>.
   The server never generates or mutates these — the operator populates them.

2. Source resolution (src/source_resolve.rs) — when pkg_install receives a
   bare slug (not a path/URL), it checks imported sources. If a source
   declares the slug, a synthetic McmPackage is built with a single ModEntry
   pointing at the source-declared download_url (external) or the source
   service's blob endpoint (source-hosted, via blob_ref).

3. install_source_mod bypasses validate_download_url (CDN allowlist) because
   imported sources are trusted — the user explicitly added them. HTTP +
   local hosts are allowed for source artifacts. Hash verification still runs
   via the download engine. Hash mismatch → "integrity check failed" (corruption),
   NOT "untrusted source" (hostile).

4. fetch_source_index (src/source_index.rs) is the shared URL-fetch+parse
   function, reused by source info, source add (indirectly), and pkg install
   source resolution. Uses no-redirect reqwest blocking client.

5. source_blob_url(index_url, blob_ref) resolves a blob reference to an
   absolute URL relative to the index URL: {index_base}/blob/{blob_ref}.
   The source service's blob route is GET /api/source/blob/{slug}.

6. No auth on source routes — sources are public catalogs (read-only index/
   meta/blob). Trust is established at import time (source add), not at read
   time.

7. tests/server.rs had 2 tests asserting the task-12 501 stubs. Updated to
   assert the real 404 "source index not configured" behavior. Same precedent
   as task 10 updating tests/mcm_package.rs stub assertions.
