Task 14: dyyl streaming host protocol design and implementation
==============================================================
Date: 2026-06-29

## Acceptance Criteria Verification

### ✅ dyyl supports --host-json or equivalent protocol
- Added `--host-json` CLI flag to `src/main.rs`
- Usage line shows: `dyyl [--debug] [--host-json] [--lang <en|zh>] <filename>`
- Binary compiled and verified: `./target/debug/dyyl --help` shows the flag

### ✅ Command event/response roundtrip
- NDJSON protocol implemented in `src/runtime/host_provider.rs`
- McmCommand: `{ type: "mcm_command", id: "1", name: "mcm.game.choose", args: ["1.21.1"], source_line: "..." }`
- McmResponse: `{ type: "mcm_response", id: "1", ok: true, value: "1.21.1" }`
- Integration test `two_mcm_commands_roundtrip` verifies two commands sent and received in order
- JSON serialization roundtrip test confirms protocol format

### ✅ MCM error sentinel propagation
- Integration test `host_error_sentinel_propagates` sends error response, dyyl receives Num(-1)
- Integration test `host_timeout_error_code` verifies host_timeout error code propagation
- Unit test `mcm_command_error_propagation` verifies RuntimeError with correct line and error info

### ✅ Unknown command failure
- Unit test `mcm_command_no_host_provider_returns_error` — mcm.* without host provider → error
- Integration test `mcm_command_without_host_errors` — mcm.* without host → Num(-1) sentinel
- Pre-existing unknown.cmd still returns Num(-1)

### ✅ mcm.game.choose scoping
- `GameChooseScope` struct in host_provider.rs tracks selected version
- Unit test `game_choose_scope_select_and_reset` — select, replace, reset
- Unit test `mcm_game_choose_updates_scope` — dispatching mcm.game.choose updates scope
- Integration test `game_choose_scope_persists_until_next_choose` — two choose commands tracked
- Scope resets on script end (env is dropped)

- 173 unit tests pass (13 new from host_provider + mcm modules)

- Pre-existing 1 failure: config::tests::invalid_toml_yields_error (not from this change)
