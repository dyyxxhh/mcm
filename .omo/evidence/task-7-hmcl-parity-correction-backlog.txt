## Known Limitations (Not Gaps — Design Constraints)

1. **Release files require deployment**: The `/release/{filename}` route is fully implemented with ALLOWED_RELEASE_FILES whitelist and path traversal protection, but release artifact files must be placed in `{data_dir}/release/` directory. In the test environment (temp data dir), the files are absent, so the endpoint correctly returns 404. This is NOT a bug — the route is functional and verified by code inspection.

2. **Install/pkg requires deployed packages**: The `/install/pkg/{slug}` route correctly returns 404 for nonexistent packages. The route delegates to Storage which requires actual package data. This is correct behavior.

3. **Mock mode is explicitly gated**: The server defaults to `AuthMode::Mock` when `MCM_AUTH_MODE` is unset. Real OIDC requires all four `MCM_OIDC_*` env vars, and the server fails clearly at startup if they're missing in real mode. This is verified by config tests and the startup log output.
