Task 21 — Java runtime discovery/install and compatibility matrix
=================================================================
Date: 2026-06-28
Status: COMPLETE (corrected: version verification added)

=== Corrective Changes ===

The original implementation accepted any Java on PATH or user-configured path
without verifying its actual version. It set `runtime.major = required`
(the MC-compatibility version) without probing what Java was actually running.

Fix added:
- `parse_java_version_output(output: &str) -> Option<JavaMajor>` — parses
  `java -version` stderr to extract Java 8/17/21. Unit-tested with 9 test
  case strings (Java 8, OpenJDK 8, Java 17, Java 21, EA version, unsupported,
  garbage, empty).
- `probe_java_version(path: &Path) -> Option<JavaMajor>` — runs `{path} -version`,
  captures stderr, delegates to `parse_java_version_output`.
- Updated `discover_java_impl`:
  * User-configured java_path: probe version → accept if matches required;
    bail with actionable error if wrong major; skip if unprobeable.
  * Managed runtime: verify via sidecar `java.version` marker (written at
    install time alongside the binary).
  * System Java: probe version → accept if matches required; silently fall
    through to install plan if wrong major.
- `install_managed_java` now writes `{bin_dir}/java.version` sidecar.
- Test helpers: `make_mock_java()` creates executable shell scripts with
  deterministic `java -version` output for testing probe behavior.
