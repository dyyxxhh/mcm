## 2. GAP ANALYSIS: CURRENT vs REQUIRED

### 2A. Protocol Format Gap

CURRENT: `parse_dyyl_to_lock()` (src/pkg_cmd.rs:356-383)
  - Reads .dyyl file as plain text line-by-line
  - Parses `mcm.op(args);` syntax via string splitting
  - Extracts ops via `strip_prefix("mcm.")` and `find('(')`
  - Arguments parsed via `parse_dyyl_args()` using comma-split and colon-split
  - All values become strings (no type distinction)
  - NO process spawning — just reads the file directly

REQUIRED: NDJSON host protocol (plan v2:199-204)
  - MCM spawns dyyl binary with `--host-json` flag
  - dyyl writes NDJSON to stdout: { "type":"mcm_command", "id":"...",
    "name":"mcm.game.choose", "args": [...], "source_line":"..." }
  - MCM responds via stdin: { "type":"mcm_response", "id":"...", "ok":true, "value":... }
  - MCM reads responses and correlates by `id`
  - 60s timeout per command (error.code="host_timeout")
  - Parallel safety support (parallel_safe field)
  - Error sentinels: { "type":"mcm_response", "id":"...", "ok":false,
    "error": {"code":"...", "message":"..."} }

### 2B. source_line Metadata Gap

CURRENT: `new_step()` (src/mcm_package.rs:446-452)
  - Always sets source_line: None
  - The LockStep struct has `source_line: Option<String>` (line 86)
  - But no code path ever populates it during build

REQUIRED: Every step from NDJSON must carry source_line
  - The NDJSON `mcm_command` event includes `source_line` field
  - MCM must propagate this to LockStep.source_line
  - Test asserts: `step.get("source_line").is_some()` for every step

### 2C. Determinism Gap

CURRENT: `parse_dyyl_to_lock()` produces deterministic output
  - Same text input → same steps (string parsing is deterministic)
  - But this is a side effect, not a protocol guarantee

REQUIRED: Determinism via host protocol
  - Same .dyyl source → dyyl produces same NDJSON events
  - MCM records events in order → deterministic v2 lock
  - Test asserts: two builds of same .dyyl produce identical JSON steps

### 2D. Argument Parsing Gap

CURRENT: `parse_dyyl_args()` (src/pkg_cmd.rs:399-416)
  - Positional args keyed by index: {"0":"dev","1":"1.21.1"}
  - Key:value pairs: {"game":"dev","version":"1.21.1"}
  - All values become strings (no number/bool/array distinction)

REQUIRED: JSON-native arguments from NDJSON
  - args field in mcm_command is a JSON array
  - Values retain JSON types (strings, numbers, booleans, arrays, objects)
  - No lossy string conversion

### 2E. Process Model Gap

CURRENT: No process spawning
  - build_dyyl reads file, calls parse_dyyl_to_lock, writes JSON
  - No dyyl binary involvement

REQUIRED: Spawn dyyl as subprocess
  - MCM must find and spawn dyyl binary
  - Communicate via stdin/stdout NDJSON streams
  - Handle process lifecycle (spawn, read, write, timeout, kill)
  - Handle dyyl crashes and non-zero exit codes


## 3. EXACT FUNCTIONS/CODE THAT MUST CHANGE

### 3A. Functions to Replace/Remove

1. parse_dyyl_to_lock() — src/pkg_cmd.rs:356-383
   Status: MUST BE REPLACED
   Reason: Text-line parser that violates the NDJSON host protocol
   Replacement: NDJSON host protocol reader that spawns dyyl process

2. parse_dyyl_args() — src/pkg_cmd.rs:399-416
   Status: MUST BE REPLACED
   Reason: Custom string parsing instead of JSON-native args
   Replacement: serde_json deserialization of NDJSON args array

3. parse_dyyl_value() — src/pkg_cmd.rs:419-425
   Status: MUST BE REMOVED
   Reason: All values coerced to strings, no JSON type preservation
   Replacement: Native JSON value handling from NDJSON

### 3B. Functions to Modify

4. build_dyyl() — src/pkg_cmd.rs:104-126
   Status: MUST BE REWRITTEN
   Current: reads file → parse_dyyl_to_lock → write JSON
   Required: reads file → spawn dyyl with --host-json → NDJSON protocol
     → collect events → build lock from events → write JSON
   Key changes:
   - Add dyyl process spawning (Command::new("dyyl").arg("--host-json"))
   - Add NDJSON line reader on dyyl stdout
   - Add NDJSON line writer on dyyl stdin (for responses)
   - Add 60s timeout per command
   - Add error handling for process failures

5. classify_mcm_op() — src/pkg_cmd.rs:386-396
   Status: MAY NEED ADJUSTMENT
   Current: maps text op names to permissions
   Required: still needed to map NDJSON command names to permissions
   Change: input may change from "game.choose" to "mcm.game.choose"
     depending on whether dyyl includes or excludes the "mcm." prefix

6. new_step() — src/mcm_package.rs:446-452
   Status: MUST ADD source_line PARAMETER
   Current: source_line always None
   Required: accept source_line: Option<String> parameter
   Or: builder pattern to set source_line after creation
