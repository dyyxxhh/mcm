Task 16: Permission model, upload validation, and version-root file/network semantics
=====================================================================================

## Implementation Summary

### 1. Path Validation (mcm_package.rs)
- Added `validate_step_dest_path()` - validates version-root relative paths
- Added `validate_lock_step_paths()` - validates all file/net/cwd paths at parse time
- Added `validate_lock_install_only()` - rejects non-install steps for server upload
- Paths reject: empty, null bytes, `..` traversal, absolute paths, backslashes, Windows-reserved components
- Validation runs automatically in `parse_mcm_lock()` (boundary parser)

### 2. Permission Matrix (mcm_package.rs)
- `StepPermission` enum: Install, Do, Full
- `is_install_permitted()` returns true only for Install variant
- `as_str()` and Display trait for human-readable permission names
- `validate_lock_install_only()` rejects any step with Do or Full permission

### 3. Server Upload Validation (server/storage/helpers.rs)
- `validate_payload()` calls `validate_install_only()` on uploaded content
- Rejects: non-install steps, v1 schema, actions, launch config, local data, secrets
- Returns 400 Bad Request with clear error message

### 4. `mcm install` Silent Strip (pkg_install.rs)
- `apply_lock()` silently skips steps where `!step.permission.is_install_permitted()`
- Only install-permitted steps execute during `mcm install`
- Do/full steps are silently removed without error

### 5. `mcm do` Full Graph Execution (pkg_install.rs)
- `do_lock()` executes ALL step types: install, do, and full
- root.system steps require additional NonBypassable confirmation
- mcm.do nested steps are skipped (requires dyyl host)
- Installs steps: game.choose, mod.install, shell.run, file.copy, file.write, net.download, config.set, game.install, pkg.install

### 6. Version-Root Resolution (pkg_install.rs)
- `VersionContext` struct tracks game.choose scope
- `game.choose` sets version root: `~/mcm/{game}/{version}`
- `file.copy`, `file.write`, `net.download`, `shell.run` (cwd) resolve paths relative to version root
- Default root from active profile's mods_dir parent

### 7. `game.choose` Scoping
- Each `game.choose` step resets the version context
- Scope persists until next `game.choose` or script end
- Version-scoped commands (file.*, net.download, config.set) use the current context's root
