### Route behavior notes

1. **`GET /`** returns 404 in share-only mode — the SPA fallback only serves when web files are present in the configured `web_dir`. For production, the reverse proxy serves the static site.
2. **`GET /health`** always returns `{"status":"ok","mode":"share"}` — the canonical health check.
3. **`GET /api/share/list`** returns the package list — currently empty on this instance.
4. **`GET /api/auth/oidc/start`** returns a mock auth URL because `MCM_AUTH_MODE` is not set to `real`. In production with OIDC env vars, it would return a real OIDC authorize URL.
5. **`GET /install`** returns the full bootstrap shell script for installing/updating MCM.
6. **`GET /install/pkg/{slug}`** returns 404 for nonexistent slugs — confirms the route is wired correctly.
