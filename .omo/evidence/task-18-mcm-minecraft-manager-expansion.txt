## Route behavior summary

| Route | Input | Response |
|-------|-------|----------|
| GET /install/pkg/sample-pkg | Existing package | 200 + text/x-shellscript script with shebang, mcm install --yes, dry-run, bootstrap |
| GET /install/pkg/nonexistent | Missing package | 404 JSON |
| GET /install/pkg/UPPERCASE | Invalid slug | 400 JSON |
| GET /install/pkg/has.dot | Invalid slug | 400 JSON |
| Script dry-run | MCM_INSTALL_DRY_RUN=true | Prints plan, exits 0 |
| Script bootstrap | mcm not in PATH | Calls /install endpoint |

### Script design (generate_pkg_script)

1. set -eu — fail on undefined vars/errors
2. SLUG='<slug>' — single-quoted; slug validated to [a-z0-9-] at boundary
3. Shell variable ${SLUG} used throughout
4. Env: MCM_PACKAGE_BASE_URL (default: https://mc.dyyapp.com), MCM_INSTALL_DRY_RUN
5. Dry-run: prints plan and exits 0
6. Bootstrap: curl -fsSL "${BASE}/install" | sh (trusted /install endpoint)
7. Delegation: mcm install "${BASE}/api/share/pkg/${SLUG}" --yes

## Security properties

1. **Slug validation at boundary**: validate_package_name() enforces [a-z0-9-]
2. **Single-quoted shell assignment**: SLUG='...' — impossible to break POSIX single quotes
3. **No bespoke package logic**: delegates entirely to mcm install --yes
4. **Trusted bootstrap**: calls back to /install (checksum-verified)
5. **No auth on route**: public; no Turnstile/admin token/OIDC
