## QA scenarios

### Happy path: .mrpack import
```
$ mcm --provider mock pkg install ./test.mrpack --yes

```
- Mod jar `mods/testmod.jar` written to game root
- Override `config/sodium.toml` copied
- Lock entry `testmod` recorded in `dev.lock.json`

## Design decisions
1. **Module split** under 250 LOC ceiling: hub (72) + types (62) + import (229) + export (111).
2. **`Cursor<&[u8]>`** type for `ZipArchive` â€” read file into `Vec<u8>`, pass `as_slice()`.
3. **Double-borrow fix** in `collect_zip_overrides`: two-pass (collect indices, then read bytes).
4. **Hash strategy**: sha512 verified in plan stage; sha256 computed from actual bytes for lock (not trusted from mcm meta).
5. **CurseForge resolution**: best-effort + warning; install does not fail on unresolvable mods.
6. **Path safety**: reuses `validate_asset_path`; validation before any write (no partial install).
7. **Size limits**: 256 MB total, 10 000 entries (constants in `modpack_import.rs`).
8. **Secret rejection**: `scan_for_secrets` bumped to `pub(crate)`, runs on both index and manifest JSON.
9. **Export**: `ZipWriter` with stored entries; copies `config/`/`shaderpacks/`/`resourcepacks/` into `overrides/`.
