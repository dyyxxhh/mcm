### What Changed
1. **src/upgrade_deps.rs** (NEW, 47 pure LOC) — `check_dependency_satisfaction` with real logic:
   - Required deps: must be in lock OR in upgrade plan → skip if missing
   - Incompatible deps: must NOT be in lock → skip if installed
   - Unknown/Embedded deps: must NOT be in lock → skip if installed (conservative)
   - Optional deps: no check (matches install semantics)

2. **src/upgrade.rs** — Refactored `build_upgrade_plan_for_game`:
   - First pass: collect items with newer versions
   - Build `planned_ids: BTreeSet<String>` from planned items
   - Second pass: filter items via `check_dependency_satisfaction(&available, &lock, &planned_ids)`
   - Removed stub `check_dependency_satisfaction` (moved to upgrade_deps.rs)

3. **src/provider/mock.rs** — Added `badmod` to `mock_projects()` so incompatible test works via CLI

4. **tests/upgrade.rs** — Updated test helpers:
   - `lock_single` now includes `depmod` (required dep of rootmod v1.0.0)
   - `lock_with_incompatible` now includes `depmod`
   - New `lock_without_dep` helper for required-dep-missing test
   - Two new tests: `required_dep_missing_skips_upgrade`, `incompatible_dep_installed_blocks_upgrade`

5. **src/lib.rs** — Added `mod upgrade_deps;`
