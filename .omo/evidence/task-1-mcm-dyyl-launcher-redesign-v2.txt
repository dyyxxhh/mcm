Task 1: Server asset roots and PM2 cwd independence
===================================================
Status: COMPLETE

## Changes Made

### src/server/config.rs
- Added `web_dir: PathBuf` field to `ServerConfig`
- Added `resolve_web_dir()` function with 3-tier resolution:
  1. MCM_WEB_DIR env var (explicit override)
  2. Walk ancestors of current_exe() looking for web/ subdirectory (up to 5 levels)
  3. Cwd-relative web/ fallback with warning
- Added `is_valid_web_dir()` helper that checks for index.html presence
- Updated Debug impl to include web_dir
- Updated from_env() to call resolve_web_dir()
- Updated test helper to include web_dir field

### src/server/mod.rs
- Added `web_dir()` accessor to ServerState
- Updated `build_router()` to use `state.web_dir().join(...)` for ServeFile paths
- Changed `spa_index()` to take `State<ServerState>` and read web_dir from config
- Updated all test helpers (__test_router_full, __test_router_with_mock_user) to include web_dir
- Added new `__test_router_with_web_dir()` test helper for explicit web_dir injection
- Added re-export of `resolve_web_dir` from config module

### src/lib.rs
- Added `pub use server::__test_router_with_web_dir` re-export

### tests/server.rs
- Added `static_files_served_regardless_of_cwd` test:
  - Builds router with explicit web_dir pointing to repo's web/ directory
  - Verifies /index.html returns 200 with correct HTML content
  - Verifies /dashboard (SPA fallback) returns 200 with index.html content
  - Uses CARGO_MANIFEST_DIR to find web/ reliably regardless of cwd

## Architecture Notes
- web_dir is always a PathBuf (non-optional), resolved at ServerConfig construction time
- resolve_web_dir() never panics - always returns a path (may be cwd-relative fallback)
- Test helper __test_router_with_web_dir() enables testing with explicit web_dir without env var race conditions
- No new dependencies added to Cargo.toml
