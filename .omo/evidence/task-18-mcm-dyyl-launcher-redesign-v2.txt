## No-secrets audit

### Audit method
grep for the following patterns in README.md:
1. (secret|password|token|api_key|credential)\s*[:=]\s*["'][^"']+["']
2. (sk-|ghp_|gho_|xoxb-|AKIA|eyJ[a-zA-Z0-9_-]+\.eyJ)
3. your-client-id|your-client-secret patterns (verified as documentation placeholders)

### Results
- Pattern 1 (secret=value literals): 0 matches
- Pattern 2 (real key prefixes): 0 matches
- Pattern 3 (placeholder text): 4 matches, all clearly user-instruction placeholders:
  - "your-client-id" in env export example
  - "your-client-secret" in env export example
  - "your-client-secret" in secret file example
  - "<your-client-id>" in ecosystem.config.js example

All four are documentation placeholders instructing operators to fill in their
own values. None resemble real credential literals.

### Env injection documentation added
New "Where to inject secrets" section with three methods:
1. PM2 environment variables (production recommended)
2. Secret files (Docker/systemd/CI)
3. Ecosystem config placeholders (with explicit "never commit" warning)
