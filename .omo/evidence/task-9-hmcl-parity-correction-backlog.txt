Task 9: game config write support — Evidence
=============================================

## Requirement
`game config <name>` must support setting version-scoped fields (java_path, jvm_args, extra_args),
not just showing them. The RED test `game_config_supports_setting_fields` must pass.

## Changes Made
- src/cli.rs: Added `GameConfigSubcommand` enum (Show/Set), changed `Config` variant to include
  optional subcommand. Added `#[arg(allow_hyphen_values = true)]` on the value field so values
  starting with `-` (like `-Xmx4G`, `--fullscreen`) are accepted.
- src/game_cmd.rs: Added `game_config_set()` method, updated dispatch in `game()` to route
  Config commands to show or set based on subcommand.
- src/lib.rs: Re-exported `GameConfigSubcommand`.

## CLI Syntax
- mcm game config <name>          → show (default, backward compatible)
- mcm game config <name> show     → show (explicit)
- mcm game config <name> set <key> <value> → write field
  - key: java_path | jvm_args | extra_args
  - value accepts any string including those starting with - or --
